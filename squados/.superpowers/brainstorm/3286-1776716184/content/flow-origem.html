Manual — Admin cria o item
O admin (ou master admin) preenche uma referência, título e nota inicial e dispara o fluxo. Ideal para atendimentos, casos únicos e trabalhos pontuais.
✓ Já funciona hoje
🔗
Integração — Via API / Webhook
Um sistema externo (Nomus ERP, e-commerce, CRM) dispara o fluxo automaticamente ao criar um pedido, OS ou registro. O SquadOS gera uma URL de webhook exclusiva por fluxo.
POST para URL gerada Nomus · N8N · Zapier

Encadeamento — Disparado por outro fluxo
Quando um fluxo anterior é concluído, ele cria automaticamente um item neste fluxo. Os dados (referência, título, notas e anexos) são herdados do item de origem.

Múltiplas origens combinadas
O fluxo aceita itens vindos de qualquer combinação das origens acima ao mesmo tempo — manual + webhook + encadeamento. Todos caem na mesma coluna inicial.
Todas as origens ativas
💡 Exemplo real Pós-venda: Quando o setor Comercial conclui a etapa "Expedição" no fluxo de Produção, o sistema cria automaticamente um item de Pós-venda com a mesma referência (ex: PA.0234). O responsável pela etapa 1 (Criação PA) já recebe a notificação com todos os dados herdados.
